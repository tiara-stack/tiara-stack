# sheet-db-schema

`src/schema.ts` is the authoritative schema for the sheet database. It is defined with
`effect-sql-schema` and consumed by `effect-sql-kit` for schema push and future migration
generation.

`src/zero/schema.ts` is generated by `effect-zero` during `pnpm build` and is intentionally
ignored by git.

Future generated Effect SQL migrations should live in `effect-sql-migrations/`.

The previous SQL files under `migrations/` were historical Drizzle artifacts. They were not the
production migration source for this package; schema changes were applied through push-style
workflows, so they were intentionally not ported to Effect SQL migrations.
