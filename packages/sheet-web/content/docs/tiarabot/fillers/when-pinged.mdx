---
title: When TiaraBot pings you
description: Match a filler-facing TiaraBot message to the immediate action.
---

| TiaraBot surface       | Do now                                                                  | Success looks like                                                                   |
| ---------------------- | ----------------------------------------------------------------------- | ------------------------------------------------------------------------------------ |
| Check-in DM or message | Verify hour/room and press **Check in** once.                           | The bot confirms, updates the list, and restores the lockdown role when configured.  |
| Team-submission reply  | Review added/skipped teams. Edit the source before Confirm or Reject.   | The reply shows the intended write; `/team list` separately reflects the live sheet. |
| Pinned room order      | Read the selected order and follow the host’s room code/instructions.   | You enter the intended room-order position and remain available to the host.         |
| Open-slots button      | Read the refreshed openings. Respond through the server’s fill process. | Mana/moni confirms any assignment separately.                                        |

Emergency and standby pings are server procedures after TiaraBot supplies schedule/check-in state. If a ping is unclear or you cannot act, contact your server’s mana/moni.

Only the original submitter can use team **Confirm** or **Reject**. Those actions—and a rollback failure—are terminal, so later edits cannot reopen the submission. `/team list` is a live sheet lookup, not a confirmation-status check.
